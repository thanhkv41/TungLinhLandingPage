# Project Instructions

Before editing code, always read:

1. `.agents/skills/payload/SKILL.md`
2. `PROJECT_SPEC.md`

Follow `PROJECT_SPEC.md` strictly.

Do not replace Payload CMS with another CMS.

Do not add ecommerce, cart, payment, Redux, or unnecessary complexity.

Keep the website simple, SEO-friendly, Vietnamese, and production-oriented.

Use the Payload reference files when implementing:

- Collections
- Fields
- Globals
- Hooks
- Queries
- Endpoints
- Access control
